Discrete Reactive Simulation Architecture

Pragma

A Pragma contains independent data associated with Attributes, and Links to Pragma, and itself has a globally unique identifier.

The system acts as though Pragma have all Attributes and Links defined for them, defaulting to null values. This allows references to not-existing or undefined values to work if a future Action defines them.

Weak Copies

A temporary copy of an existing Pragma, and associated with a Context. The system creates the copies automatically for two scenarios.

  • Action Processing - when an Action executes, the system creates a weak copy to write the new values. The original Pragma keeps a set of Commits open while the action processing occurs. As actions complete, the weak copy closes and allows the Commit to execute on the weak copy.
  • Predictive Analysis - a Function defined as Predictive constructs a new Context in which all affected Pragma run. These Pragma never commit back to the source Pragma.

Strong Copies

Creates a full new copy of an existing Pragma. This includes correct duplication of the back links and a unique ID. This allows for cloning existing objects for permanent storage in the system.

Because of the way weak copies work, a Strong Copy of a Pragma has no commit phase into the system. The system adds it directly into the system, either with an automatically generated ID (guaranteed to have no conflicts with existing Pragma), or a well defined ID which may result in a soft error.

Commit

Weak copies of Pragma schedule themselves to commit changes to their Attributes back to the original Pragma. The system can do this on a per-Attribute basis, allowing for heavy parallelization.
